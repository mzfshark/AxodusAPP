# Axodus Academy Design System & UX Specification

## Vision

Axodus Academy is not a traditional LMS.

It is:

- governance-native educational infrastructure
- Proof-of-Knowledge operational layer
- Learn-to-Win onboarding system
- credential and treasury-aware learning network
- DAO-aligned educational economy

The Academy must feel like:

- institutional learning infrastructure
- operational certification platform
- governance onboarding system
- economic participation gateway

It must not feel like:

- gamified mobile course apps
- influencer-course marketplaces
- casual e-learning portals
- crypto hype platforms

The visual identity must align with the Axodus ecosystem standards defined by the global design system and token architecture.

---

## Core UX Philosophy

The Academy exists to:

- onboard participants into the Axodus ecosystem
- validate knowledge through governance-aware systems
- distribute educational incentives
- certify operational competency
- create DAO-aligned contributors

Every interface must reinforce:

- progression
- legitimacy
- educational accountability
- transparent reward distribution
- operational readiness

---

## Design Language

### Visual Direction

The Academy should visually resemble:

- Stripe Learn
- Linear
- Notion Enterprise
- Datadog Docs
- Safe Global
- Coursera Enterprise
- GitBook
- enterprise certification systems

Avoid:

- neon-heavy crypto visuals
- influencer-course aesthetics
- exaggerated gamification
- childish UX
- meme-oriented UI

Use:

- dense operational layouts
- modular educational panels
- certification dashboards
- governance-linked progress systems
- infrastructure-oriented composition

---

## Official Design Tokens

The Academy must reuse the official Axodus token architecture.

Primary tokens:

- Background: `--color-bg`
- Surface Primary: `--color-surface-primary`
- Surface Secondary: `--color-surface-secondary`
- Accent: `--color-accent`
- Success: `--color-success`
- Warning: `--color-warning`
- Error: `--color-error`

Typography:

- Inter
- JetBrains Mono

JetBrains Mono must be used for:

- certificate IDs
- wallet addresses
- reward hashes
- governance metadata
- treasury identifiers
- Proof-of-Knowledge references

- educational infrastructure

- compact
- operational
- enterprise-grade

## Page Architecture

### Main Layout Structure

Recommended structure:

- global top navigation
- DAO and identity context
- learning navigation sidebar
- content workspace
- certification and reward panels
- governance context panels

Academy pages must render inside the global AxodusAPP shell and must not duplicate the global header, footer, or primary ecosystem navigation.

## Primary Academy Sections

### Public

#### Landing

Purpose:

- ecosystem educational entrypoint
- explain Learn-to-Win
- explain Proof-of-Knowledge
- expose institutional credibility

#### Course Catalog

Purpose:

- discover learning tracks
- explore certifications
- filter by governance level
- filter by operational domain

#### Instructor / Organization Profiles

Purpose:

- validate legitimacy
- expose governance standing
- show certification quality

### Authenticated

#### Dashboard

The Dashboard is the operational home.

Must expose:

- active learning tracks
- reward progression
- certification status
- governance eligibility
- DAO-linked permissions
- treasury rewards
- pending validations

Dashboard should resemble:

- operational workspace
- contributor portal
- certification console

#### Learning Workspace

The learning workspace is the core Academy experience.

Must support:

- video lessons
- text lessons
- quizzes
- governance checkpoints
- Proof-of-Knowledge validations
- milestone unlocks
- live certification state

Avoid distraction-heavy layouts.

Learning must feel:

- serious
- structured
- accountable

#### Certification Center

The Certification Center must expose:

- earned certifications
- DAO-recognized credentials
- governance eligibility
- operational permissions
- certification expiration
- verification hashes
- public verification links

Status examples:

- active
- pending-validation
- suspended
- revoked
- under-review

#### Reward & Treasury Panel

The Academy is tied to token emission.

Users must clearly understand:

- earned rewards
- locked rewards
- unlocked rewards
- pending milestones
- vesting schedules
- DAO reward origin
- treasury source

Never present rewards like casino winnings.

Rewards must feel like:

- educational grants
- contributor incentives
- governance-backed issuance

#### Governance Eligibility

Educational progression impacts governance access.

This panel should expose:

- governance level
- DAO eligibility
- contributor standing
- operational reputation
- proposal permissions
- validator eligibility
- committee eligibility
