# AxodusAPP Wallet

## Wallet Philosophy

Wallet interaction is the foundation of ecosystem participation.

Wallet UX must remain:

- transparent
- predictable
- secure
- chain-aware
- DAO-aware

---

# Core Responsibilities

The wallet layer must manage:

- connection state
- chain state
- signatures
- transactions
- session persistence
- provider compatibility

---

# Supported Wallets

Initial direction:

- MetaMask
- WalletConnect
- Rabby
- Coinbase Wallet
- injected EVM wallets

---

# Supported Chains

The app should support multichain interaction.

Supported chains must be configurable through registry systems.

---

# Wallet UX Rules

Always show:

- active wallet
- active chain
- transaction status
- required network
- DAO execution context

---

# Transaction UX

Before submission:

- explain action
- explain risks
- explain costs
- explain permissions
- explain governance implications

---

# Error Handling

The app should clearly explain:

- rejected signatures
- unsupported chains
- failed transactions
- RPC issues
- wallet disconnects

---

# Long-Term Goal

Create a reliable multichain wallet experience suitable for governance, treasury operations, learning systems, marketplace activity, and ecosystem execution.
